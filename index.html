Первым этапом - минимально жизнеспособный продукт (MVP) - Планировщик посещений. Возможные сценарии дальнейшего развития.

На основе сформированных на первом этапе баз данных по:

  • студентам
  • сотрудникам
  • расписанию с фиксацией посещений
добавить базу с результатами по учебным дисциплинам.
На основе MVP и сформированных баз реализовать:
  1. Печать необходимых материалов для контрольных по спискам записавшихся.
  2. Журнал успеваемости по проверенным контрольным.
  3. Онлайн-версия зачетки для контроля за сдачей предметов.
  4. ...
Кроме того, появится возможность установления прямого контакта школы с родителями при минимальных усилиях со стороны родительских комитетов. Уведомления о собраниях, мероприятиях, взносах и пр.
